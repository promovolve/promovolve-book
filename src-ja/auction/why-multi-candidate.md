# なぜ複数候補なのか？

スロットごとに複数の候補を保持し、単一のオークション勝者を選ばないという判断は、Promovolveにおいて最も重要なアーキテクチャ上の選択です。

## 単一勝者オークションの問題

従来の広告エクスチェンジでは、各オークションが1人の勝者を生み出します:

1. **搾取の罠**: 最高入札者が常に勝利し、CTRがひどくても同様
2. **探索がない**: より低い入札のクリエイティブがより良いパフォーマンスを示すかどうかを発見するメカニズムがない
3. **脆弱な配信**: 勝者の予算が尽きた場合、システムは再オークションするか何も表示しないかのいずれか
4. **インセンティブの不整合**: エクスチェンジは収益を最適化し、ユーザー体験を最適化しない

## 複数候補がこれをどう解決するか

Promovolveの公平な選択アルゴリズムはキャンペーンごとの多様性を保証し（まず各キャンペーンに1つのクリエイティブ、次に残りを埋める）、配信時にThompson Samplingがそれらの中から探索を行います。

### スコアリング計算式

```
score = sampledCTR × log(1 + CPM)
```

`sampledCTR`は、時間バケット化された統計（1分粒度、60分ローリングウィンドウ）を使用して`Beta(clicks + 1, non_clicks + 1)`から抽出されます。

### 探索の実際の動作

```
Slot candidates after fair selection:
  Campaign A: CPM $5.00, Beta(6, 146)    → sample: 0.032
  Campaign B: CPM $4.20, Beta(3, 19)     → sample: 0.091
  Campaign C: CPM $3.80, Beta(1, 1)      → sample: 0.647

Scores:
  A: 0.032 × log(6.00) = 0.032 × 1.79 = 0.057
  B: 0.091 × log(5.20) = 0.091 × 1.65 = 0.150
  C: 0.647 × log(4.80) = 0.647 × 1.57 = 1.016

→ C wins (exploration of unknown creative)
```

### グレースフルデグラデーション

Campaign Aが予算を使い切った場合:
1. ペーシングゲートがThompson Samplingの**前に**予算をチェック
2. Campaign Aがフィルタリングされる
3. Thompson SamplingはBとCのみで実行される
4. 再オークション不要 — DData操作なし
5. Aの予算が翌日リセットされると、配信が再開される（エントリはServeIndexに保持されている）

### パブリッシャーとの利害の一致

`sampledCTR`ファクターは、ユーザーが実際にクリックするクリエイティブを自然に優遇します。高CPMだが低CTRのクリエイティブは、時間の経過とともにエンゲージメントの高いクリエイティブに負けていきます。これにより、パブリッシャーの利益（エンゲージメント、ユーザーの信頼）と広告主の利益（実際のクリック）が一致します。

## トレードオフ

複数候補選択は、最高入札者が常に勝つわけではないことを意味します。これは短期的なCPM収益を減少させますが、以下を増加させます:

- **長期的な収益**: 高いCTR → より多くのクリック → より良いキャンペーンROI → 広告主の高いリテンション
- **システムの耐障害性**: フォールバック候補が再オークション頻度を削減
- **学習**: Thompson Samplingは探索率のチューニングなしで最適なパフォーマーに収束
