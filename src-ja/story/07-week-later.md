# 第7章: 1週間後

7日が経過した。システムが何を学習したか見てみよう。

## タケシのRL Agentが戦略を持つ

1日目、agentのepsilonは0.92で、ほぼすべてのアクションがランダムだった。7日目までに、epsilonは約0.15に減衰した。agentは85%が活用、15%が探索になっている。

学習したポリシーはQ-valuesに見える:

**予算が60%以上残っていて正午前の場合**: 積極的に入札する（action 4: 1.2倍）。ユキのサイトの朝のトラフィックはトラベルコンテンツで高いCTRを持つ。これらのインプレッションを獲得するためにより多く支払う価値がある。

**予算が30%以下で時間が残っている場合**: 引く（action 0: 0.8倍）。残りを節約する。午後に薄く広げる方が、早く燃え尽きるよりも合計クリック数が多くなる。

**CTRが高く、支出レートが正常な場合**: 維持する（action 2: 1.0倍）。うまくいっている。壊れていないものを変えない。

**オーバーペーシング（支出レートが1.5倍以上）の場合**: 即座に削減する（action 0または1）。報酬関数のoverspendペナルティがこの反応を訓練した。agentはオーバーペーシングが負の報酬につながることを学習した。

これらのルールはどれもプログラムされていない。報酬シグナル（クリック - overspendペナルティ）とreplay bufferでの数千回の訓練ステップから生まれた。

一方、JR Rail Pass agentは異なる戦略を学習した。より高いCPM ($8) とより大きな予算により、朝に積極的に入札して午後はペーシングに任せる余裕がある。multiplierはより大きく振れる: 朝のピークで1.4まで上昇し、夕方は0.7まで低下する。高いベースCPMがあるため、0.7倍でもほとんどのオークションで勝てることを学習した。

異なるキャンペーン、異なる予算、異なる学習された戦略。各agentが自分自身の状況に適応する。

## Thompson Samplingが収束した

数百のインプレッション後、クリエイティブ統計が明確なストーリーを語る:

| クリエイティブ | インプレッション | クリック | 分布 | 平均CTR |
|----------|------------|--------|-------------|----------|
| タケシの旅館 | 312 | 14 | Beta(15, 299) | 4.8% |
| JR Rail Pass | 287 | 8 | Beta(9, 280) | 3.1% |
| Hiking Gear Co | 89 | 1 | Beta(2, 89) | 2.2% |
| 陶芸ワークショップ | 45 | 3 | Beta(4, 43) | 8.5% |

タケシの旅館が最もインプレッションを獲得している。実証されたCTRとまずまずのCPMがあるからだ。JR Rail Passはより高いCPMだがより低いCTRだ。スコアリング公式`sampledCTR × log(1 + CPM)`により競争力は維持されるが、タケシのCTRの優位性が効いている。

陶芸ワークショップは興味深い。インプレッション数は少ない（週の後半に開始した）が、CTRは最も高い8.5%だ。`Beta(4, 43)`の分布はまだかなり広い。Thompson Samplingはこの高いCTRが本物かノイズかを確認するために、より多くの探索を与えている。

ハイキング用品広告はほぼフェードアウトした。`Beta(2, 89)`はほとんどの場合ゼロ近くをサンプリングする。インプレッションの約5%を得ている。何かが変化した場合（新しいクリエイティブ、季節の変化）を検知するのに十分な探索だけだ。もし広告主がより良いクリエイティブをアップロードすれば、システムは数時間以内に改善を検知する。

これらの配分の決定は誰も行っていない。誰もハイキング用品キャンペーンを一時停止したり、陶芸ワークショップをブーストしたりしていない。システムが純粋な学習を通じて正しい配分を見つけた。

## Category Rankerが意見を持つ

ユキのサイトのTaxonomyRankerEntityが1週間のデータを蓄積した:

```
Travel:           Beta(45, 355)  — 11.3% CTR, tight distribution
Hiking/Camping:   Beta(8, 192)   — 4.0% CTR, fairly confident
East Asian Culture: Beta(5, 45)  — 10.0% CTR, still exploring
Food & Drink:     Beta(2, 28)    — 6.7% CTR, early data
```

Travelが優勢だ。ほとんどのオークションで最も高い重みを得る。しかしEast Asian Cultureが予想外の好パフォーマーだ。陶芸ワークショップ（East Asian Cultureカテゴリ）がこれを牽引している。ランカーはEast Asian Cultureにより多くのオークション重みを与えており、そのカテゴリの広告主により多くの入札機会が生まれることを意味する。

これが好循環を生む: 良いカテゴリパフォーマンス → より多くのオークション重み → より多くの候補 → より多くのデータ → より良いThompson Sampling → より良い広告 → より高いCTR → より高いカテゴリパフォーマンス。

## トラフィック形状が調整された

TrafficShapeTrackerがユキのサイトの7日間の時間帯別データを保持し、`dayAlpha = 0.2`でブレンドされている:

```
Weekday profile:
  Hour  0-6:   1-2% each  (late night, minimal traffic)
  Hour  7:     4%          (morning commute)
  Hour  8-10: 10-12% each (peak reading time)
  Hour 11-13:  6-7% each  (lunch)
  Hour 14-17:  3-4% each  (afternoon lull)
  Hour 18-20:  7-9% each  (evening reading)
  Hour 21-23:  3-4% each  (winding down)
```

PI controllerは線形の時間割合の代わりにこの形状を使用する。午前10時に、通常1日のトラフィックの32%が通過したことを知っている（線形と仮定した場合の42%ではない）。つまり午前10時のペーシングターゲットは「予算の約32%を使っているべき」であり、42%ではない。結果: 予算は架空の一様分布ではなく、1日の実際のトラフィックパターンに沿って正しく配分される。

## ペーシングが自己調整した

PI controllerが自身を調整してきた:

- **Overpace multiplier**: 2.0倍からスタート。タケシのキャンペーンが朝に時々20%オーバーペースすること（RL agentが入札を上げる）を検出し、2.8倍に引き上げた。これはcontrollerが過剰支出に対してより積極的に反応することを意味する。経験から学んだ修正だ。

- **Spend ratio smoothing**: 適応的EMAのalphaが0.25に落ち着いた。ユキのサイトのトラフィックは中程度の変動がある（新しい記事を公開してソーシャルメディアに投稿するとスパイクする）。controllerはこれらのスパイクに過剰反応しないように、デフォルトよりも多く平滑化することを学習した。

## ユキが見るもの

ユキがパブリッシャーダッシュボードを確認する:

This Week:

  Revenue:             $9.24
  Active advertisers:  4

収益は人生を変えるほどではない。小さなブログだ。しかし広告は関連性があり、サイトは高速で、読者からの苦情もない。彼女は2つの保留中のクリエイティブ（京都ウォーキングツアーと日本語学校）を承認し、次の記事の執筆に戻る。

## タケシが見るもの

タケシが広告主ダッシュボードを確認する:

```
This Week:
  Impressions:  312
  Clicks:       14
  CTR:          4.5%
  Spend:        $1.56
  Avg CPC:      $0.11
```

14人がトラベルブログから彼の旅館の予約ページにクリックスルーした。まさに彼がリーチしたかった読者だ。クリック単価は$0.11。入札を管理したり、ターゲティングを調整したり、DSPのインターフェースを学んだりする必要はなかった。写真をアップロードし、予算を設定すれば、システムが残りをやってくれた。

彼は1日の予算を$30に増額する。

## システムは学習し続ける

8日目が始まる。Epsilonが0.05に減衰する。RL agentのフロアだ。これ以降、95%のアクションが学習されたポリシーに従い、5%が探索を続ける。agentは学習を終えたわけではない。季節の変化、新しい競合、トラフィックの変動に適応していく。しかし最初の1週間の激しいランダム探索は終わった。

システムはそのリズムを見つけた: 関連性のある広告が掲載されたトラベルブログ、興味を持つ旅行者にリーチする地元の旅館、継続的に最適化されるクリエイティブパフォーマンス、スムーズにペーシングされる予算。すべてがサブミリ秒の配信で、ユーザートラッキングなしに稼働している。

これが、ユーザーではなくコンテンツから始めた場合の広告の姿だ。
