# 第5章: クリック

読者がタケシの旅館広告をクリックする。ブラウザがクリックURLをたどる:

```
GET /v1/click?pub=yuki-site&url=...&cid=ryokan-001&tok=a8f3...&rid=01HX...
```

この1回のクリックが、システム全体にわたる学習の連鎖を引き起こす。

## クリック検証

TrackRoutesハンドラがクリックを検証する:

1. **HMAC検証**: `tok`パラメータが、パブリッシャーのシークレット、URLパラメータ、リクエストIDから計算された署名と照合される。誰かがURL（クリエイティブID、キャンペーン、CPM）を改ざんした場合、署名が一致しない。`403 Forbidden`。

2. **鮮度チェック**: `b`パラメータがタイムバケット（1分粒度）をエンコードしている。クリックURLが3分以上古い場合、拒否される。これによりリプレイウィンドウが制限される。

3. **リプレイガード**: 正規URL（一意のリクエストIDを含む）がシャーディングされたbloom filterと照合される。この正確なクリックが既に記録されている場合、重複だ。`409 Conflict`。

3つすべてが通過。クリックは正当だ。`204 No Content` — 確認済み。

## 4つのシステムがこのクリックから学習する

`LearningEventLog`がクリックイベントをシステムの4つの異なる部分にルーティングし、それぞれが異なるタイムスケールで学習する。

### 1. TaxonomyRankerEntity — カテゴリランキング（日単位）

クリックイベントがユキのサイトのTravelのランカーに到達する:

```
Before: Beta(12, 88)  — mean CTR: 12%
After:  Beta(13, 88)  — mean CTR: 12.9%
```

分子に1クリック追加。ユキのサイトにおけるTravelカテゴリのスコアがわずかに上昇する。数週間かけて、これがこのサイトのオークションでどのカテゴリが優先されるかを形成する。一貫してクリックを獲得するカテゴリは上昇し、そうでないカテゴリは衰退する。

ランカーは7日間の半減期減衰を使用する。古いインプレッションとクリックは徐々に重みを失う。これにより、ランキングは季節の変化に適応する: 人々が旅行を計画する秋にはTravelが優勢になり、ホリデーダイニングを検索する12月にはFood & Drinkが上昇するかもしれない。

### 2. AdServer — クリエイティブThompson Sampling（分単位）

クリックがAdServer上のタケシのクリエイティブ統計に到達する:

```
Before: Beta(1, 1)   — uniform, no data
After:  Beta(2, 1)   — heavily skewed toward high CTR
```

1インプレッション、1クリック。100%のクリック率だ。当然これは続かないが、タケシのクリエイティブに強い初期シグナルを与える。Beta distributionは`Beta(2, 1)`となり、ほとんどの場合高い値をサンプリングする。次の数インプレッションでは、このクリエイティブがThompson Samplingで優遇される。

さらに20インプレッションと1クリックの後、`Beta(3, 19)`になる。約14%のCTRだ。まだ良いが、より現実的だ。分布は真実に向かって狭まっている。

統計は1分バケットの60分ローリングウィンドウを使用する。このクリエイティブの強い初期パフォーマンスは次の1時間の配信判断に影響を与え、その後データがエージングアウトし始め、システムは変化に対して反応性を保つ。

### 3. CampaignEntity — RL Agent（15分単位）

クリックがRL agentのウィンドウカウンターに記録される:

```
windowClicks: 0 → 1
```

これは即座にRL actionをトリガーしない。agentは15分ごとにしか観察しないからだ。しかし次の`RLObserveTick`が発火した時、このクリックは報酬の一部となる:

```
reward = windowClicks - overspendPenalty = 1 - 0 = 1.0
```

正の報酬だ。agentはこの遷移を保存する:「状態S（フル予算、午前、multiplier 1.0）にいて、入札を維持するアクション（action 2）を選択し、報酬1.0を得た。」何百ものこうした遷移を通じて、agentはどの状態とアクションがクリックにつながるかを学習する。

### 4. ダッシュボードプロジェクション（秒単位）

クリックはトラッキングジャーナルに書き込まれる。これはイベントをバッチ処理してPostgreSQLに書き込むバッファされたPekko Streamだ。数秒以内に、タケシの広告主ダッシュボードが更新される:

```
Impressions: 1  |  Clicks: 1  |  CTR: 100%  |  Spend: $0.005
```

当然、データが増えるにつれてこれらの数値は正常化する。しかしタケシは、キャンペーンがライブでエンゲージメントを得ていることを確認できる。

## 複合効果

これは1回のクリックだ。しかし何に触れたかに注目してほしい:

| システム | 学習した内容 | タイムスケール | 効果 |
|--------|----------------|-----------|--------|
| Category Ranker | Travelはこのサイトで効果的 | 日〜週 | Travel広告がオークションでより高い重みを得る |
| Creative Stats | このクリエイティブはクリックされる | 分〜時間 | Thompson Samplingが優遇する |
| RL Agent | この入札レベルはクリックを生む | 15分ウィンドウ | Bid multiplierが日単位で調整される |
| Dashboard | キャンペーンはパフォーマンスを発揮している | 秒 | 広告主が結果を確認する |

各システムはそれぞれのペースで学習する。Thompson Samplingは数分以内に反応する。このクリックのおかげで、次の読者には異なる広告ミックスが表示されるかもしれない。RL agentは数時間以内に反応する。bid multiplierは1日の終わりまでにシフトするかもしれない。Category rankerは数週間かけて反応する。ユキのサイトでのTravelの重みが徐々に増加する。

5つの学習レイヤー、すべて1回のクリックから、5つの異なるタイムスケールで。手動の最適化なし。「入札を調整しよう」もなし。システムは自力で正しい答えに収束していく。

## 一方、もう一つのクリエイティブ

タケシの旅館広告がクリックを獲得する一方、ハイキング用品広告は3インプレッションでクリックゼロだ。その分布は`Beta(1, 4)`となった。平均CTRは約20%だが、分布は低い値に傾き始めている。

Thompson Samplingはまだ時々これを選択する。`Beta(1, 4)`の分布は0から0.8のどこでもサンプリングできるが、高い値をサンプリングする確率が低いだけだ。もし次のインプレッションでクリックを獲得すれば、即座に回復する: `Beta(2, 4)`ははるかに競争力のある分布だ。

クリックが得られない状態が続けば、自然にフェードアウトする。50インプレッションで0クリック、つまり`Beta(1, 51)`になる頃には、サンプルはほぼ常にゼロ近くになる。誰も停止の判断をすることなく、事実上表示されなくなる。

これがThompson Samplingの美しさだ: 効果の低いクリエイティブは手動で一時停止する必要がない。自ら消滅していく。
