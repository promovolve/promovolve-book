# 第1章: パブリッシャーが参加する

ユキは京都でトラベルブログを運営している。寺院、ハイキングコース、季節の祭り、隠れた名店について記事を書いている。読者は関西への旅行を計画している人たちで、熱心で好奇心旺盛、各記事にじっくりと時間をかけて読んでくれる。週に2〜3本の記事を丁寧に調べて公開している。

彼女はサイトを台無しにせずに収益化したいと思っている。ポップアップは嫌だ。自動再生の動画広告も嫌だ。コンテンツと全く関係のない「昨日スニーカーを見ましたよね」というリターゲティング広告も嫌だ。彼女が求めているのは、サイトに馴染む広告だ。嵐山の旅館、ハイキング用品店、地域の鉄道パスなど。

彼女はPromovolveに登録し、サイトに小さなJavaScriptスニペットを追加する。サイドバーに300×250のレクタングル広告枠と、記事セクション間に728×90のバナー広告枠の2つを作成する。

初日に彼女がやることはこれだけだ。ここからはシステムが引き継ぐ。

## クローラーが起動する

午前2時、Promovolveのクローラーがユキのサイトを訪問する。Playwrightベースのヘッドレスブラウザで、JavaScriptをレンダリングし、ページをスクロールし、表示されているテキストコンテンツを抽出する。また、ユキが配置した広告枠を検出し、そのサイズと位置を記録する。

クローラーはホームページからリンクをたどり、最近の記事まで設定可能な深さ（デフォルト: 2階層）まで巡回する。ユキのブログの場合、ホームページ、記事一覧ページ、そして各個別記事が対象となる。

各ページについて、クローラーは表示されているテキスト（記事本文、見出し、キャプション）をキャプチャする。この生テキストが、ページの内容を理解するためにシステムが使用するものだ。

## LLMがコンテンツを分類する

クローラーが取得したテキストはLLMに送られる。デフォルトではGoogleのGemini Flashが使われており、コスト面で選ばれている（1回の呼び出しあたり1セントの何分の1か）。プロンプトはモデルに対し、コンテンツをIAB Content Taxonomy 2.1カテゴリに分類するよう指示する。これは広告業界がページの内容を記述するための標準語彙だ。

ユキの記事「東京都東部の紅葉ハイキング」に対して、LLMは以下を返す:

```json
{
  "selected_taxonomy_ids": [
    {"id": "596", "confidence": 0.95},
    {"id": "564", "confidence": 0.85},
    {"id": "483", "confidence": 0.70}
  ]
}
```

つまり: Travel (596) が高い確信度、Hiking/Camping (564)、そしてEast Asian Culture (483)。これでシステムは、このページが何について書かれているかを、広告主が理解できる言語で把握した。

LLMがダウンしている場合、circuit breakerが5回連続の失敗後にトリップし、30秒間リクエストを停止する。そのページはこのクロールサイクルでは分類されないだけだ。次回のクロールで処理される。クラッシュもなく、配信パスの劣化もない。

## カテゴリランキング: このサイトで何が効果的か？

分類はストーリーの終わりではない。ユキのサイトではすべてのカテゴリが同じパフォーマンスを発揮するわけではない。「Travel」広告は4%のクリック率かもしれないが、「East Asian Culture」広告は0.5%かもしれない。時間の経過とともに、システムはこれを学習する。

各(カテゴリ, サイト)ペアには`TaxonomyRankerEntity`がある。これは、特定のサイトにおけるそのカテゴリのクリックスルーパフォーマンスのBeta distributionを保持する小さなactorだ。初期段階ではデータがないため、すべてのカテゴリは`Beta(1, 1)`から始まる。つまり完全な無知状態だ。インプレッションとクリックが蓄積されるにつれて、分布は鋭くなる。

ランカーはThompson Sampling（配信時にも使われる同じアルゴリズム。詳しくは後述）を使用して各カテゴリに重みを割り当てる。実績のあるパフォーマンスが高いカテゴリは高い重みを得る。データの少ないカテゴリは変動する重みを得る。高い時もあり（探索）、低い時もある。

重みの上位3カテゴリがオークションに転送される。ある程度の履歴があるユキのサイトでは、Travel（高い実証済みCTR）、Hiking/Camping（まずまずのCTR）、そしてFood & Drinkのような新しいカテゴリ（不確実だが探索する価値あり）になるかもしれない。

## ページはオークションの準備完了

この時点で、Promovolveは以下を把握している:
- **ページの内容**: Travel、Hiking、East Asian Culture（LLMから）
- **このサイトでどのカテゴリが効果的か**: TravelとHiking（ランカーから）
- **利用可能な広告枠**: 300×250サイドバー、728×90バナー（クローラーから）

この情報は、ユキのサイトの`AuctioneerEntity`に保存される。これはサイトIDでシャーディングされたactorで、各URLの最新の分類を記憶している。ページは広告主が入札できる状態になった。

ユーザー側ではまだ何も起きていない。読者の表示速度が低下することもない。広告が表示されることもない。オークションのインフラは、ユキと読者が眠っている間に、スケジュールに従ってバックグラウンドで構築された。
