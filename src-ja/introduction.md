# 技術的イントロダクション

この章では、Promovolve のアーキテクチャとアルゴリズムの簡潔な技術概要を提供する。これらの設計選択の背景にある動機については、[なぜ Promovolve なのか？](./why-promovolve.md)を参照してほしい。

## 5つの主要メカニズム

### 1. 定期バッチオークション

オークションは、ページロードのたびにではなく、コンテンツが公開または更新されたとき（スケジュールされたクロール + 5分間隔の再オークション）に実行される。LLM がページコンテンツを IAB カテゴリに分類し、TaxonomyRankerEntity がサイト固有のパフォーマンスに基づいてカテゴリをランク付けし、CategoryBidderEntity が適格なキャンペーンから入札を収集する。結果は DData にキャッシュされる。

### 2. マルチ候補キャッシング

単一のオークション勝者ではなく、広告枠ごとに複数の候補がキャンペーンごとの多様性保証付きでショートリストされ、ServeIndex（DData 経由でレプリケートされるインメモリ）に格納される。これにより、オークションを再実行することなく、配信時の探索が可能になる。

### 3. 配信時の Thompson Sampling

ユーザーがページを読み込むと、Thompson Sampling がキャッシュされた候補の中から選択する：

```
score = sampledCTR × log(1 + CPM)
```

CTR は、時間バケット統計（1分粒度、60分ローリングウィンドウ）を使用した Beta-Bernoulli 事後分布からサンプリングされる。`log(1 + CPM)` の項により、高い入札には優位性があるが CTR が支配的となる — ユーザーが実際にクリックするクリエイティブは、単に高く入札しただけのクリエイティブに勝つ。

### 4. Double DQN 入札最適化

各キャンペーンは、キャンペーンごとの強化学習エージェント（8→64→64→5 ニューラルネットワーク）を実行し、15分ごとにパフォーマンスを観察して入札倍率を調整する。エージェントは、複数日のエピソードにわたってクリック最大化と予算ペーシングのバランスを学習する。

### 5. 自己チューニング PI ペーシング

適応的ゲイン、トラフィック形状学習（平日/週末別の24時間プロファイル）、振動検出、リーキー積分器アンチワインドアップを備えた PI コントローラーが、予算配分を平滑化する。トラフィックが午前10時にピークを迎え午後3時に落ち込むことを学習し、自動的に調整する。

## その結果

サブミリ秒の広告配信。5つのレイヤーでの継続的学習（リクエストごとの Thompson Sampling、15分間隔の RL、オークションごとのカテゴリランキング、日次のトラフィック形状、継続的な PI チューニング）。予算が枯渇した際のグレースフルデグラデーション。ユーザートラッキングなし。すべてのクリエイティブに対するパブリッシャーの承認。
