# 報酬関数

報酬関数は、フロアCPMエージェントが何を最適化するかを定義します。`FloorCpmOptimizationAgent.scala`より：

```
reward = normalizedRevenue × fillRate
       − emptyPenalty
       − budgetExhaustionPenalty
       − volatilityPenalty
       − boundaryPenalty
```

## Revenue × Fill Rate

主要なシグナル。収益はベースライン（フロア価格での1インプレッション）で正規化され、絶対的なフロアレベルに依存しません。

フィルレートとの乗算は、収益がゼロの場合でもグラディエントシグナルを提供します。これなしでは、エージェントはフロアが高すぎる場合に「収益ゼロ」しか見えず、*どれだけ悪いか*の指標がありません。フィルレートがそのグラディエントを提供します：
- フロアがわずかに高すぎる → fillRate=0.6 → 部分的な報酬
- フロアが大幅に高すぎる → fillRate=0.0 → ゼロの報酬

## 空オークションペナルティ

```
emptyPenalty = weight × max(0, 0.5 − fillRate)
```

フィルレートが50%を下回ると発動。フィルレートが低下するにつれてペナルティが線形に増加します。

## 予算枯渇ペナルティ（weight = 2.0）

```
budgetExhaustionPenalty = 2.0 × budgetExhaustionRate
```

最も重要なペナルティ。高いフロアは予算をより早く消耗させます（単独勝者はフロア価格を支払う）。このペナルティは収益がゼロに落ちる前にそのカスケードを捕捉します。

## ボラティリティペナルティ

```
volatilityPenalty = 0.3 × |currentFloor − prevFloor| / prevFloor
```

激しい変動を抑制します。広告主は価格の安定性を必要とします。ペナルティは相対的な変化に比例するため、$5フロアでの$0.10の変化（2%）は、$0.50フロアでの$0.10の変化（20%）よりも低くペナルティされます。

## 設計原則

報酬関数は意図的にシンプルです。多くの項を持つ複雑な報酬関数はデバッグが困難で、予期しない局所最適を生み出す可能性があります。各項には明確な目的があります：

- Revenue × fillRate：「パブリッシャーの収入を最大化」
- 空ペナルティ：「オークションを空にしない」
- 予算ペナルティ：「広告主の予算を枯渇させない」
- ボラティリティペナルティ：「安定を保つ」
- 境界ペナルティ：「限界値で固着しない」
