# Thompson Samplingをゼロから理解する

旅行ブログに3つの広告があるとします。どの広告が読者に好まれるかは分かりません。最も効果の低い広告に何千ものimpressionを無駄にせずに、どうやって最適な広告を見つけ出せるでしょうか？

これが**Multi-Armed Bandit**問題であり、Thompson SamplingがPromovolveの解決策です。この章ではゼロから直感的な理解を構築します。

## 問題：探索と活用のトレードオフ

カジノに3台のスロットマシンがあると想像してください。それぞれ異なる（未知の）当たり率を持っています。手持ちは1,000コインです。どうすれば勝ちを最大化できるでしょうか？

- **純粋な活用**：最初に当たりが出たマシンをずっと使い続ける。問題：たまたま当たっただけかもしれない。他のマシンの方が良い可能性がある。
- **純粋な探索**：3台すべてを均等に、それぞれ333回ずつプレイする。問題：そのマシンが悪いと分かった後も、最悪のマシンにコインを無駄にしている。

最適な戦略はこの中間にあります。早い段階で探索してどのマシンが最良かを学び、次第に最良のマシンの活用にシフトしていくのです。これが**探索と活用のトレードオフ**です。

Promovolveでは、「スロットマシン」は広告クリエイティブです。「当たり」はユーザーがクリックするかどうかです。「コイン」はimpression――それぞれが広告主にとってのコストであり、パブリッシャーの広告枠を消費します。最もパフォーマンスの良いクリエイティブをほとんどの場合に表示したいですが、新しい広告やまだ不確実な広告も試して、より良いものがないか学ぶ必要があります。

## 良くない解決策：A/Bテスト

「どのクリエイティブが最良か？」に対する標準的なアプローチはA/Bテストです。各クリエイティブを同数のユーザーに表示し、統計的に有意な差が出るまで待ち、勝者を宣言してそれを全員に表示します。

これは機能しますが、無駄が多いです。クリエイティブAのクリック率が5%、クリエイティブBのクリック率が0.5%の場合、均等に分割するとテスト期間全体を通じてトラフィックの半分が悪いクリエイティブを見ることになります。そしてテスト終了後は学習が止まります。新しいクリエイティブが来たら、新しいテストを始める必要があります。

本当に必要なのは、次のようなシステムです：
1. 各クリエイティブを数回試す
2. どれが良いかを素早く判断する
3. 良いクリエイティブにトラフィックを移行する
4. 完全に試すのをやめない――クリエイティブのパフォーマンスが変化する場合に備えて

Thompson Samplingはこれらすべてを自動的に行います。

## 核心的アイデア：不確実性による探索

核心となる洞察はこうです。単一の数値（「クリエイティブAのクリック率は5%」）を追跡する代わりに、その数値に対する**不確実性**を追跡するのです。

2回のimpressionと1回のクリック後、クリエイティブAのクリック率は約50%だと考えられますが、あまり確信はありません。10%から90%のどこかかもしれません。

200回のimpressionと10回のクリック後、クリック率は約5%だと考えられ、かなり確信があります。おそらく3%から8%の間です。

Thompson Samplingはこの不確実性を直接利用します。**各クリエイティブの不確実性分布からランダムな値をサンプリングし、最も高いサンプル値を持つクリエイティブを選択します。**

データの少ないクリエイティブは広い分布を持ち、高くサンプリングされることも低くサンプリングされることもあります。そのため、時々試されます（探索）。データの多いクリエイティブは真のクリック率を中心とした狭い分布を持ちます。実際のパフォーマンスに近い値で一貫してサンプリングされます（活用）。

探索は不確実性に比例して自然に発生します。チューニングパラメータも、明示的な探索/活用の切り替えも不要です。

## Beta Distribution：クリック率のモデリング

クリック率は確率、つまり0から1の間の数値です。**Beta distribution**は確率に対する不確実性を表現する自然な方法です。

Beta distributionには2つのパラメータがあります：**α（alpha）**と**β（beta）**。

- **α**は「成功」（クリック）の回数に事前分布を加えたもの
- **β**は「失敗」（クリックなしのimpression）の回数に事前分布を加えたもの

`Beta(1, 1)`——一様分布、つまり「全く分からない、0%から100%のどのクリック率も同じくらいありえる」——から始めて、各観測で分布を更新します：

```
Start:       Beta(1, 1)         — uniform, total ignorance
1 click:     Beta(2, 1)         — probably high CTR, but uncertain
1 no-click:  Beta(2, 2)         — back toward 50%, still very uncertain
8 no-clicks: Beta(2, 10)        — probably low CTR (~17%), getting more sure
2 clicks:    Beta(4, 10)        — ~29%, narrowing
```

Beta(α, β)の**平均**は`α / (α + β)`です。しかしThompson Samplingは平均を使いません。分布からランダムな値を**サンプリング**します。それがこの手法を機能させる鍵です。

## 具体例

旅行ブログの広告枠にキャッシュされた3つのクリエイティブ。いくつかのimpressionの後：

| Creative | Impressions | Clicks | Distribution | Mean CTR |
|----------|------------|--------|-------------|----------|
| A (hotel ad) | 150 | 6 | Beta(7, 145) | 4.6% |
| B (tour ad) | 20 | 2 | Beta(3, 19) | 13.6% |
| C (new airline ad) | 0 | 0 | Beta(1, 1) | 50.0% |

読者がページを読み込みます。Thompson Samplingが各クリエイティブから1つのランダムサンプルを抽出します：

```
A: sample from Beta(7, 145)  → 0.038  (probably near its true ~5%)
B: sample from Beta(3, 19)   → 0.091  (wide distribution, sampled somewhat high)
C: sample from Beta(1, 1)    → 0.647  (uniform distribution, sampled very high)
```

クリエイティブCがこのラウンドで勝ちます。65%のクリック率があると考えているからではなく、Cについて何も分かっておらず、たまたまサンプルが高く出たからです。これが探索です。

もしCのCTRが低いと判明すれば、20回のimpressionの後に分布は狭まり（例えばBeta(2, 19)）、高いサンプル値が出なくなります。もしCが実際に優れたCTRを持っていれば、分布は高い位置を維持し、より多くのimpressionを獲得します。

**システムは「テストを開始する」「テストを終了する」と決定することなく、真実に収束します。**

## スコアリング：CTRと入札価格の組み合わせ

クリック率だけが重要なわけではありません。パブリッシャーは収益も重視します。CTR 2%でCPM $8のクリエイティブは、CTR 3%でCPM $2のものより価値が高いかもしれません。

Promovolveのスコアリング式は両方をバランスさせます：

```
score = sampledCTR × log(1 + CPM)
```

なぜ単なる`CPM`ではなく`log(1 + CPM)`なのか？

2つのクリエイティブを考えてみましょう：
- A: $2 CPM, 4% CTR → score = 0.04 × log(3) = 0.04 × 1.10 = 0.044
- B: $10 CPM, 1% CTR → score = 0.01 × log(11) = 0.01 × 2.40 = 0.024

対数関数はCPMの範囲を圧縮します。5倍の入札をしても5倍のスコアにはなりません。約2倍のスコアになります。つまり、クリエイティブが一貫して勝つためには、実際に良いパフォーマンス（高いCTR）が必要です。ひどい広告で単に入札額を上げるだけでは勝てません。

これはパブリッシャーと広告主のインセンティブを一致させます。パブリッシャーは収益だけでなく、エンゲージメントのある読者も獲得できます。最高額入札者のお金だけではありません。

## Cold Start：新しいクリエイティブの扱い

クリエイティブのimpressionがゼロの場合、その分布は`Beta(1, 1)`——一様分布です。0から1のどこからでもサンプリングできます。これにより自然な探索ブーストが得られますが、ランダムです。

Promovolveは構造化されたcold start戦略を追加しています：

**Full cold start**（すべてのクリエイティブのimpressionが0）：オークションからのcategory scoreを事前分布として使用し、小さなランダムノイズを加えます。これによりコンテンツ分類からブートストラップします。旅行ページ上の旅行広告は、妥当な推定値から始まります。

**Warmup**（すべてのクリエイティブのimpressionが10未満）：impression数によるround-robin。impressionが最も少ないクリエイティブが次に選ばれます。これにより、Thompson Samplingが引き継ぐ前に、すべてのクリエイティブが最低限の試行回数を確保します。

**Partial cold start**（新しいクリエイティブと確立されたクリエイティブの混合）：新しいクリエイティブは30%の探索ブーストを受けます。サンプル値が低くてもepsilon-greedy確率で選択されます。

**Steady state**（すべてのクリエイティブのimpressionが10以上）：純粋なThompson Sampling。分布は良い判断を下すのに十分な情報を持っています。

これらの戦略は`ThompsonSampling.scala`、特に`select()`メソッドにあります。

## 時間バケット統計

クリック率は時間とともに変化します。先週パフォーマンスが良かった広告が、今は陳腐化しているかもしれません。Promovolveは**1分間の粒度**を持つ**60分間のローリングウィンドウ**を使用しています：

- impressionとクリックは1分間のバケットに記録される
- スコアリング時には直近60個のバケットのみがカウントされる
- 古いデータは自動的に除外される

これにより：
- クリエイティブのパフォーマンスが変化した場合、1時間以内にシステムが適応する
- 今朝は良かったが午後には悪くなったクリエイティブは素早く修正される
- Beta distributionは常に最近の関連データに基づく

## パイプラインにおけるThompson Samplingの位置

Thompson Samplingはリクエストごとに実行されるわけではありません。いくつかのゲートの後ろに位置しています：

```
Request arrives
  → Content recency check (is the page fresh enough?)
  → Frequency cap check (has this user seen this ad too many times?)
  → Rate tracking (record this request for pacing)
  → Pacing gate (PI controller: should we serve or skip?)
  → Thompson Sampling (which creative to show?)
  → Budget reservation (does the winning campaign have budget?)
  → Serve the ad
```

pacing gateはThompson Samplingの**前に**実行されます。これは重要です。pacingコントローラーがこのリクエストをスキップすると判断した場合（予算を節約するため）、Thompson Samplingは実行されません。これにより、スロットルされたリクエストで探索が無駄になることを防ぎます。

## Thompson Samplingが他のアプローチより優れている理由

**Epsilon-Greedy（ε-greedy）との比較**：90%の確率で最良のクリエイティブを表示し、10%の確率でランダムに表示します。シンプルですが、探索率は固定で適応しません。10,000回表示したクリエイティブも10回しか表示していないクリエイティブと同じ探索率になります。Thompson Samplingは不確実なクリエイティブをより多く自然に探索します。

**UCB（Upper Confidence Bound）との比較**：`平均 + 信頼ボーナス`が最も高いクリエイティブを選びます。決定論的で、同じ状態なら常に同じクリエイティブを選びます。Thompson Samplingのランダム性は利点です。同じページを同時に読み込んだ2人のユーザーが異なるクリエイティブを見る可能性があり、多様なデータをより速く生成します。

**A/Bテストとの比較**：固定配分、固定期間、手動セットアップ。Thompson Samplingは連続的、適応的、自動的です。
