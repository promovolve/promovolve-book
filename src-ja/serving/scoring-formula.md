# スコアリング式

配信時、Thompson Samplingがどのクリエイティブを表示するかを選択します。スコアはクリエイティブの品質と広告主の入札を組み合わせます：

```
score = sampledCTR × CPM^α
```

ここで：
- **sampledCTR**：クリエイティブのBeta事後分布からのランダムな抽出 — 読者がクリックする可能性
- **CPM**：広告主の1000インプレッションあたりの最大入札額
- **α**：パブリッシャーの入札ウェイト — お金と品質のどちらをどれだけ重視するか

## パブリッシャーのダイヤル

指数αはパブリッシャーの最も重要なコントロールです。品質とお金が対立した場合に誰が勝つかを決定します：

| 設定 | α | 効果 |
|------|---|------|
| ディスカバリー | 0.3 | 品質が支配的。優れたCTRの$2の広告が、平凡なCTRの$5の広告に勝つ。広告主基盤の成長に適する。 |
| バランス | 0.5 | 均等な重み。スコア = CTR × √CPM。デフォルト。 |
| レベニュー | 0.7 | お金が支配的。高い入札が勝ちやすい。短期的な収益を最大化。 |

広告主が少ない小規模パブリッシャーはディスカバリーを選びます — 全員を競争力のある状態に保ち、品質の高いクリエイティブを勝たせ、より多くの広告主を引き付けます。高トラフィックのニュースサイトはレベニューを選びます — 各インプレッションから最大限の価値を引き出します。

## なぜlog(1 + CPM)ではないのか？

初期バージョンでは`log(1 + CPM)`で入札差を圧縮していました。これは圧縮が強すぎました：

```
log:  $10入札 / $1入札 → 3.5倍のスコア差のみ
sqrt: $10入札 / $1入札 → 3.2倍の差
linear: $10入札 / $1入札 → 10倍の差
```

`log`では、$5の入札が$1の入札をかろうじて上回る程度でした。わずかなCTRの差が5倍の入札差を覆すことができました。これはパブリッシャーに偏りすぎていました — より多く入札する広告主がほとんど恩恵を受けませんでした。

設定可能な`CPM^α`により、パブリッシャーがトレードオフを選択できます。α=0.5（sqrt）では、10倍の入札差が3.2倍のスコア差になります — 意味のある差ですが圧倒的ではありません。品質は依然として重要です。

## 数値例

2つのキャンペーンが旅行記事の広告枠を競います。パブリッシャーはバランス（α=0.5）を使用：

| | 竹志の旅館 | JRレールパス |
|---|---|---|
| 最大CPM | $5.00 | $8.00 |
| 真のCTR | ~4% | ~2% |
| サンプルCTR | 0.038 | 0.025 |
| CPM^0.5 | 2.24 | 2.83 |
| **スコア** | **0.085** | **0.071** |

竹志の旅館が38%少ない入札にもかかわらず勝利。1.5倍のCTR優位がJRの1.26倍の入札優位（sqrt圧縮後）を上回りました。

パブリッシャーがレベニュー（α=0.7）に切り替えた場合：

| | 竹志の旅館 | JRレールパス |
|---|---|---|
| CPM^0.7 | 3.09 | 4.60 |
| **スコア** | **0.118** | **0.115** |

ほぼ互角 — JRのお金がほぼ追いつきます。αをもう少し上げればJRが勝ちます。

## 品質調整型価格

勝者は最大CPMを支払いません。そのCTRを考慮した上で**勝つために必要な最低入札額**を支払います：

```
支払い = (最高敗者スコア / 勝者CTR) ^ (1/α)
```

上記の例（バランス、α=0.5）では：
- 旅館がスコア0.085で勝利
- JRの敗者スコア：0.071
- 旅館の支払い：(0.071 / 0.038)² = $3.49

竹志は$5.00で入札しましたが$3.49を支払います。品質ディスカウントで30%節約。これはシステムが優れたクリエイティブ制作を報酬するものです — CTRが高いほど実効コストが低くなります。

エクスプロレーション時（コールドスタート、ウォームアップ、インプレッションシェア保証）、勝者はパブリッシャーのフロア価格を支払います。学習目的で与えられたインプレッションに市場価格を支払うべきではありません。

## コールドスタートバリアント

クリエイティブのインプレッションがゼロの場合、Thompson Samplingはクリックからの CTR を推定できません。代わりに、TaxonomyRankerEntityからのカテゴリレベルスコアをプライアとして使用します：

```
sampledCTR = categoryScore + random(-0.15, +0.15)
score = sampledCTR × CPM^α
```

±0.15のノイズにより、コールドな候補にもエクスプロレーションのための分散が確保されます。

## 広告主が見るもの

広告主はダッシュボードで2つのCPM値を確認できます：
- **最大CPM**：入札額（$5.00）
- **eCPM**：実際に支払う平均額（例：$3.41）

この差が品質ディスカウントです。優れたクリエイティブ（高CTR）を持つ広告主は入札額よりも大幅に少なく支払います。低品質のクリエイティブを持つ広告主は最大に近い額を支払います。インセンティブが整合します：より良い広告を作れば、より少なく支払う。
