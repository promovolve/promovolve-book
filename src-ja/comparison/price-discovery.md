# 価格発見 & First-Priceモデル

従来のexchangeは価格発見のために**セカンドプライスオークション**を使用します。Promovolveは**RL調整付きのファーストプライス**を使用します。

## 従来：セカンドプライスオークション

```
Bids: $8.00, $5.50, $3.20
Winner pays: $5.51 (second-highest + $0.01)
```

**正直な入札**：過大入札してもコストが増加しないため、入札者は真の価値で入札します。
**価格発見**：市場均衡価格（$5.51）がインプレッションの価値を明らかにします。

実際には、DSPはセカンドプライスの仕組みを利用するためにbid-shadingを使用しており、多くのexchangeはすでにファーストプライスに移行しています。

## Promovolve：ファーストプライス、RL調整付き

広告主は`maxCpm`を設定します。DQNエージェントが入札乗数で調整します：

```scala
bidCpm = max(maxCpm × bidMultiplier, floorCpm)
```

広告主はこのCPMを正確に支払います — セカンドプライスの割引はありません。

### なぜファーストプライスが許容されるか

1. **単一パブリッシャープラットフォーム**：正直な開示を必要とする競争的なexchangeのダイナミクスがない
2. **RLが最適化を処理**：DQNエージェントが日数をかけて適切な入札レベルを学習（5つのアクション：0.8xから1.2x、累積的に適用）
3. **シンプル**：セカンドビッドの追跡、リザーブプライス、bid-shading対策が不要
4. **Thompson Samplingが「勝利」を再定義**：高いCPMだけでは勝てない — クリエイティブは良いCTRも持つ必要がある

### 価格最適化としてのRL

DQNエージェントの報酬関数（クリック - 過剰支出ペナルティ）が自然に効率的な入札レベルを発見します：
- 過払いの場合（高CPC、spendRate > 1.5）：報酬が低下 → エージェントが乗数を引き下げ
- 入札不足の場合（低い勝率、少ないインプレッション）：エージェントが乗数を引き上げ
- 予算内でクリックを最大化する入札レベルに収束

## 市場均衡価格なし

Promovolveは市場均衡価格を発見**しません**：
- 広告主はビジネス判断に基づいて`maxCpm`を設定
- RLは価格発見ではなくペーシングを最適化
- 広告主に競合と比較して過払い/過少払いであることを伝えるメカニズムがない

これは、価格設定が市場の問題ではなくビジネスの意思決定である**内部広告**には許容されます。

## 比較

| 側面 | Second-Price (RTB) | First-Price (Promovolve) |
|--------|-------------------|-------------------------|
| 支払価格 | 2位の入札 + $0.01 | 正確に入札額 |
| 最適化 | DSP bid-shading | DQN RLエージェント |
| 発見 | 市場均衡価格 | なし |
| 複雑性 | 高い | 低い |
| Floor price | `floorCpm`（デフォルト$0.50） | 同じ |
