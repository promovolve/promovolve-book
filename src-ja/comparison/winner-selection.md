# 勝者選択：MAB vs 最高入札

従来のexchangeは単一の勝者 — 最高入札者 — を選んで終わりです。Promovolveは根本的に異なるアプローチを取ります：オークション時に複数の候補をショートリストに入れ、配信時にThompson Samplingを使ってどのクリエイティブが実際に最も優れたパフォーマンスを発揮するかを学習します。

## 従来：最高入札者が勝利

標準的なad exchangeでは、勝者選択はシンプルです：

```
Bids received:    $8.00,  $5.50,  $3.20
Winner:           $8.00
Price paid:       $5.51   (second-price: winner pays $0.01 above second bid)
```

ロジックは純粋に金銭的です：最も多く支払う意思のある者がインプレッションを獲得します。クリエイティブの品質、ページへの関連性、クリックされる可能性は意思決定に一切関与しません。

### なぜこれが問題か

**CTRが見えない。** CPM 8ドルでクリックスルー率0.5%のキャンペーンが、CPM 3ドルでCTR 5%のキャンペーンに勝ちます。パブリッシャーはより悪い広告を配信し、クリックあたりの収益が少なくなります。より良いクリエイティブを持つ広告主は、読者により多くの価値を提供しているにもかかわらず敗れます。

**学習メカニズムがない。** exchangeは落札したクリエイティブがクリックされたかどうかを追跡しません。8ドルの入札が価値があったかどうかを知りません。各オークションは独立しており、システムは賢くなりません。

**勝者の呪い。** 競争的なオークションでは、最高入札者は統計的にインプレッションの価値を最も過大評価した者です。洗練されたDSPはこれを考慮しますが、小規模広告主はそうせず、過払いします。

**新規参入者が競争できない。** 潜在的に優れたクリエイティブを持つが控えめな入札額の新しい広告主は、決して勝たず、インプレッションを得ず、したがって自分を証明する機会を得ることがありません。システムには探索がなく、今日最も高く入札する者の活用のみです。

## Promovolve：2フェーズの選択システム

Promovolveは勝者選択を2つのフェーズに分割します：オークション時（コンテンツがクロールされたとき）の**公平なショートリスト作成**と、配信時（ユーザーが到着したとき）の**適応的選択**です。この分離が重要な設計上の違いです。

### フェーズ1：オークション時の公平なショートリスト作成

単一の勝者を選ぶ代わりに、AuctioneerEntityはキャンペーンごとの多様性保証付きで広告枠ごとに複数の候補をショートリストに入れます：

```
3 campaigns, 3 slots → each campaign gets exactly 1 slot
2 campaigns, 3 slots → each gets 1, fill the 3rd with the best remaining creative
4 campaigns, 3 slots → top 3 by CPM each get 1 slot
```

アルゴリズム：
1. すべての候補をCPM降順でソートし、**事前承認されたクリエイティブを優先**するタイブレーカーを使用（パブリッシャーはオークションに入る前にクリエイティブを承認でき、承認済みクリエイティブは未承認のものに対してタイで勝つ）
2. キャンペーンごとにグループ化し、キャンペーンごとに最良のクリエイティブを選択
3. スロットよりキャンペーンが多い場合、CPM上位のキャンペーンがそれぞれ1スロットを獲得
4. スロットよりキャンペーンが少ない場合、すべてのキャンペーンに代表権が保証され、残りのスロットは次善のクリエイティブで埋められる

**なぜこれが重要か：** 従来のexchangeでは、単一の高予算キャンペーンがすべてのインプレッションを独占できます。Promovolveの多様性保証により、参加しているすべてのキャンペーンが候補プールで代表権を得て、Thompson Samplingに学習するための多様なセットが提供されます。

### フェーズ2：配信時のThompson Sampling

ユーザーがページを読み込むと、Thompson Samplingがショートリストされた候補の中から選択します。スコアリングの式：

```
score = sampledCTR × log(1 + CPM)
```

`sampledCTR`は60分のローリングウィンドウで観察されたパフォーマンスに基づくBeta分布から抽出されます：

```
sampledCTR ~ Beta(clicks + 1, impressions - clicks + 1)
```

`log(1 + CPM)`の項は高い入札に有利に働きますが、**収穫逓減**があります：CPM 10ドルはCPM 1ドルの3.5倍しか有利ではありません（10倍ではなく）。これはCTRが支配的であることを意味します — 読者が実際にクリックするクリエイティブが、単に高く入札するだけのものに勝ちます。

### 具体例

同じスロットを競う3つのキャンペーン。Campaign Cはデータのない新規です：

```
Campaign A: $5.00 CPM, 150 impressions, 5 clicks
  Beta(6, 146) → sample: 0.032
  score = 0.032 × log(6.00) = 0.057

Campaign B: $4.20 CPM, 22 impressions, 3 clicks
  Beta(4, 20) → sample: 0.091
  score = 0.091 × log(5.20) = 0.150

Campaign C: $3.80 CPM, 0 impressions, 0 clicks
  Beta(1, 1) → sample: 0.647  (uniform — could be anything)
  score = 0.647 × log(4.80) = 1.016  ← wins (exploration)
```

Campaign Cは最も低いCPMで実績がないにもかかわらず、このリクエストで勝利します。これが**探索**です — システムが新しいクリエイティブに自分を証明する機会を与えます。次の数十インプレッションで、Cの真のCTRが低いことが判明すれば、そのBeta分布は狭まり、勝たなくなります。Cが本当に優れていることが判明すれば、安定したインプレッションシェアを獲得します。

従来のexchangeはCampaign Cを配信することはありません。データを得ることもありません。チャンスを得ることもありません。

### コールドスタート：新しいクリエイティブの立ち上げ

Thompson Samplingが機能するにはデータが必要なため、Promovolveは候補プールの状態に応じて新しいクリエイティブに対する特定の戦略を使用します：

| 条件 | 戦略 | 動作 |
|-----------|----------|----------|
| すべての候補がインプレッション0 | **フルコールドスタート** | `categoryScore ± noise`を推定CTRとして使用 — オークションのコンテンツ関連性シグナルが最初の選択をブートストラップ |
| すべての候補がインプレッション10未満 | **ウォームアップラウンドロビン** | 常に最もインプレッションが少ない候補を配信 — 活用が始まる前にすべてのクリエイティブが最低10インプレッションを取得することを保証 |
| 一部の候補は新規、一部にはデータあり | **部分コールドスタート** | 30%の確率でランダムに新しいクリエイティブを選択；70%はすべての候補に対して通常のThompson Samplingを実行 |
| すべての候補がインプレッション10以上 | **標準** | 完全なThompson Sampling |

部分コールドスタートの30%探索率は意図的に積極的です — 新しいクリエイティブは迅速にデータを必要とし、十分なインプレッションを得た後はThompson Samplingの自然な探索が残りを処理します。

### 完全な選択パイプライン

Thompson Samplingは単独で実行されるわけではありません。パイプラインの1ステップであり、パイプライン内のその位置は意図的です：

```
1. ServeIndex lookup     → fetch cached candidates from local DData replica
2. Content recency       → drop candidates if page content is stale (> 48h)
3. Frequency cap         → drop candidates the user has seen too many times
4. Pacing gate           → probabilistic throttle based on aggregate budget utilization
5. Thompson Sampling     → score and select among remaining candidates
6. Budget reservation    → reserve spend with the selected campaign
```

**なぜペーシングがThompson Samplingの前に実行されるか：** ペーシングが選択後に実行されると、Thompson Samplingがクリエイティブを選び、その後ペーシングがときどきそれを破棄します。これは探索の機会を無駄にします — 何も表示せず、何も学習しません。ペーシングゲートを先に置くことで、Thompson Samplingに到達するすべてのリクエストが配信されたインプレッションと有用なデータを生成します。

### 予算予約と優雅なフォールバック

Thompson Samplingが勝者を選択した後、システムは支出の予約を試みます：

```
1. Reserve spend with CampaignEntity
2. Verify budget status with AdvertiserEntity
3. On failure (budget exhausted) → try the next-best candidate by Thompson score
4. All candidates exhausted → return NoCandidates (HTTP 204)
```

従来のexchangeでは、勝者が支払えない場合、オークションは失敗し、スロットは空になります（または低品質のフォールバック広告が表示されます）。Promovolveの複数候補モデルでは、常に次善の選択肢が待っています — オークションを再実行することなく優雅にデグレードします。

## 比較表

| 次元 | 従来（最高入札） | Promovolve（Thompson Sampling） |
|-----------|--------------------------|-------------------------------|
| **選択基準** | 価格のみ | CTR × log圧縮価格 |
| **候補数** | 1人の勝者 | スロットごとに複数のショートリスト |
| **学習** | なし — 各オークションは独立 | 継続的 — すべてのインプレッションがBeta事後分布を更新 |
| **新規クリエイティブ発見** | 既存の最高入札者を上回らない限り不可能 | 探索により組み込み；新規クリエイティブには30%の強制探索 |
| **コールドスタート** | 新しい広告主は勝つために高く入札する必要がある | ラウンドロビンウォームアップが初期データ収集を保証 |
| **障害処理** | スロットが空になる | 次善の候補にフォールスルー |
| **パブリッシャーとの整合性** | 広告主の支出を最適化 | 読者のエンゲージメント（CTR）を最適化、支出で重み付け |
| **短期収益** | 高い（常に最高入札を選択） | 時に低い（低CPMのクリエイティブを探索） |
| **長期収益** | 停滞（学習なし） | 高い（より多くのクリックを稼ぐ高CTRクリエイティブを発見） |
| **広告主ROI** | 大きな予算を優遇 | クリエイティブの質を優遇 — CPM 3ドルの良い広告がCPM 8ドルの悪い広告に勝てる |
